Ref Hook Documentation ​

A comprehensive guide to using the Ref hook in Olova Framework

Introduction ​

The Ref hook provides a way to hold mutable values that persist across renders without causing re-renders when the value changes. It's commonly used to access DOM elements directly and store values that shouldn't trigger component updates.

Basic Syntax ​

jsx
import { $ref } from "olova";

const refContainer = $ref(initialValue);
// Access value using: $refContainer.current

Core Concepts ​

Key Characteristics ​

  • Mutable - current value can be changed without causing re-renders
  • Persistent - Value persists between renders
  • No Re-render - Changing $ref value doesn't trigger component re-render
  • Single Property - Always access value through .current

Basic Examples ​

DOM Element $reference ​

jsx
import { $ref, Effect } from "olova";

const FocusInput = () => {
  const inputRef = $ref(null);

  Effect(() => {
    // Focus the input element when component mounts
    inputRef.current?.focus();
  }, []);

  return (
    <input
      ref={inputRef}
      type="text"
      placeholder="I will be focused on mount"
    />
  );
};

Video Player Control ​

jsx
import { $ref } from "olova";

const VideoPlayer = ({ src }) => {
  const videoRef = $ref(null);

  const handlePlay = () => {
    videoRef.current?.play();
  };

  const handlePause = () => {
    videoRef.current?.pause();
  };

  return (
    <div>
      <video ref={videoRef} src={src} />
      <button onClick={handlePlay}>Play</button>
      <button onClick={handlePause}>Pause</button>
    </div>
  );
};

Advanced Usage ​

Storing Previous Values ​

jsx
import { $ref, Effect, State } from "olova";

const Counter = () => {
  const [count, setCount] = State(0);
  const prevCountRef = $ref(count);

  Effect(() => {
    prevCountRef.current = count;
  }, [count]);

  return (
    <div>
      <p>Current: {count}</p>
      <p>Previous: {prevCountRef.current}</p>
      <button onClick={() => setCount((c) => c + 1)}>Increment</button>
    </div>
  );
};

Interval Management ​

jsx
import { $ref, Effect, State } from "olova";

const Timer = () => {
  const [count, setCount] = State(0);
  const intervalRef = $ref(null);

  const startTimer = () => {
    if (intervalRef.current) return;
    intervalRef.current = setInterval(() => {
      setCount((c) => c + 1);
    }, 1000);
  };

  const stopTimer = () => {
    if (intervalRef.current) {
      clearInterval(intervalRef.current);
      intervalRef.current = null;
    }
  };

  // Clean up interval on unmount
  Effect(() => {
    return () => stopTimer();
  }, []);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={startTimer}>Start</button>
      <button onClick={stopTimer}>Stop</button>
    </div>
  );
};

Form Validation ​

jsx
import { $ref, State } from "olova";

const FormWithValidation = () => {
  const [isValid, setIsValid] = State(false);
  const formRef = $ref(null);

  const validateForm = () => {
    const form = formRef.current;
    if (!form) return;

    const isFormValid = form.checkValidity();
    setIsValid(isFormValid);
  };

  return (
    <form ref={formRef} onChange={validateForm}>
      <input type="text" required minLength="3" placeholder="Required field" />
      <button disabled={!isValid}>Submit</button>
    </form>
  );
};

Canvas Drawing ​

jsx
import { $ref, Effect } from "olova";

const Canvas = () => {
  const canvasRef = $ref(null);
  const isDrawing = $ref(false);

  Effect(() => {
    const canvas = canvasRef.current;
    const ctx = canvas.getContext("2d");

    const startDrawing = (e) => {
      isDrawing.current = true;
      draw(e);
    };

    const stopDrawing = () => {
      isDrawing.current = false;
      ctx.beginPath();
    };

    const draw = (e) => {
      if (!isDrawing.current) return;

      const rect = canvas.getBoundingClientRect();
      const x = e.clientX - rect.left;
      const y = e.clientY - rect.top;

      ctx.lineTo(x, y);
      ctx.stroke();
      ctx.beginPath();
      ctx.moveTo(x, y);
    };

    canvas.addEventListener("mousedown", startDrawing);
    canvas.addEventListener("mousemove", draw);
    canvas.addEventListener("mouseup", stopDrawing);
    canvas.addEventListener("mouseout", stopDrawing);

    return () => {
      canvas.removeEventListener("mousedown", startDrawing);
      canvas.removeEventListener("mousemove", draw);
      canvas.removeEventListener("mouseup", stopDrawing);
      canvas.removeEventListener("mouseout", stopDrawing);
    };
  }, []);

  return <canvas ref={canvasRef} width="500" height="500" />;
};

Best Practices ​

1. Initialization ​

jsx
// ✅ Good: Initialize with proper default value
const valueref = $ref(null);
const countref = $ref(0);
const optionsref = $ref({ setting: true });

// ❌ Bad: Undefined initialization
const valueRef = $ref();

2. DOM Manipulation ​

jsx
// ✅ Good: Check for ref existence
const MyComponent = () => {
  const elementRef = $ref(null);

  const doSomething = () => {
    if (elementRef.current) {
      elementRef.current.scrollIntoView();
    }
  };

  return <div ref={elementRef}>Content</div>;
};

3. Callback Refs ​

jsx
const MeasureComponent = () => {
  const [height, setHeight] = State(0);

  const measuredRef = (node) => {
    if (node !== null) {
      setHeight(node.getBoundingClientRect().height);
    }
  };

  return (
    <div ref={measuredRef}>
      <p>Height: {height}px</p>
    </div>
  );
};

Common Pitfalls ​

1. State vs Ref ​

jsx
// ❌ Bad: Using ref for rendered values
const Counter = () => {
  const countRef = $ref(0);

  return (
    <button
      onClick={() => {
        countRef.current += 1; // Won't trigger re-render
      }}
    >
      Count: {countRef.current}
    </button>
  );
};

// ✅ Good: Using state for rendered values
const Counter = () => {
  const [count, setCount] = State(0);

  return <button onClick={() => setCount((c) => c + 1)}>Count: {count}</button>;
};

2. Mutating Ref Objects ​

jsx
// ⚠️ Be careful when mutating ref objects
const Component = () => {
  const dataRef = $ref({ count: 0 });

  // ✅ Good: Create new object reference
  const updateData = () => {
    dataRef.current = { ...dataRef.current, count: dataRef.current.count + 1 };
  };

  // ❌ Bad: Direct mutation might cause issues with object references
  const mutateData = () => {
    dataRef.current.count += 1;
  };
};

Use Cases ​

  1. DOM References

    • Form inputs
    • Canvas elements
    • Media elements (video, audio)
    • Scroll containers

  2. Instance Values

    • Timers and intervals
    • Animation frames
    • Previous value tracking
    • Event listeners

  3. Third-Party Library Integration

    • Chart libraries
    • Map components
    • Rich text editors
    • Animation libraries

Remember that Refs should be used sparingly and primarily for values that:

  • Need to persist between renders
  • Shouldn't trigger re-renders when changed
  • Require direct DOM manipulation

For values that should trigger re-renders when changed, use State instead.
