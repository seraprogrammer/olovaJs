$effect Hook Documentation ​

A comprehensive guide to using the $effect hook in Olova Framework

Introduction ​

The $effect hook in Olova allows you to perform side $effects in functional components. Side $effects might include data fetching, subscriptions, manual DOM manipulations, or any code that needs to interact with the outside world.

Basic Syntax ​

jsx
import { $effect } from "olova";

$effect(
  () => {
    // $effect code here

    // Optional cleanup function
    return () => {
      // Cleanup code here
    };
  },
  [
    /* dependencies */
  ]
);

Core Concepts ​

$effect Timing ​

  • $effects run after every render by default
  • Cleanup functions run before the next $effect execution and component unmount
  • Dependencies array controls when $effects re-run

Dependencies Array ​

  • Empty array ([]) - $effect runs once after initial render
  • No array - $effect runs after every render
  • Array with values - $effect runs when dependencies change

Basic Examples ​

Data Fetching ​

jsx
import { $effect, State } from "olova";

const UserProfile = ({ userId }) => {
  const [user, setUser] = State(null);
  const [loading, setLoading] = State(true);
  const [error, setError] = State(null);

  $effect(() => {
    const fetchUser = async () => {
      try {
        setLoading(true);
        const response = await fetch(`/api/users/${userId}`);
        const data = await response.json();
        setUser(data);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchUser();
  }, [userId]); // Re-run when userId changes

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;
  if (!user) return null;

  return (
    <div>
      <h1>{user.name}</h1>
      <p>{user.email}</p>
    </div>
  );
};

Event Listeners ​

jsx
import { $effect } from "olova";

const WindowSize = () => {
  const [size, setSize] = State({
    width: window.innerWidth,
    height: window.innerHeight,
  });

  $effect(() => {
    const handleResize = () => {
      setSize({ width: window.innerWidth, height: window.innerHeight });
    };

    window.addEventListener("resize", handleResize);

    // Cleanup: remove event listener
    return () => {
      window.removeEventListener("resize", handleResize);
    };
  }, []); // Empty array means run once on mount

  return (
    <div>
      Window size: {size.width} x {size.height}
    </div>
  );
};

Advanced Usage ​

Cleanup Functions ​

jsx
import { $effect, State } from "olova";

const ChatRoom = ({ roomId }) => {
  const [messages, setMessages] = State([]);

  $effect(() => {
    const connection = createConnection(roomId);
    connection.connect();

    connection.on("message", (message) => {
      setMessages((prev) => [...prev, message]);
    });

    // Cleanup function
    return () => {
      connection.disconnect();
    };
  }, [roomId]); // Reconnect when roomId changes

  return (
    <div>
      {messages.map((msg) => (
        <div key={msg.id}>{msg.text}</div>
      ))}
    </div>
  );
};

Multiple $effects ​

jsx
import { $effect, State } from "olova";

const UserDashboard = ({ userId }) => {
  const [user, setUser] = State(null);
  const [posts, setPosts] = State([]);
  const [notifications, setNotifications] = State([]);

  // Fetch user data
  $effect(() => {
    fetch(`/api/users/${userId}`)
      .then((res) => res.json())
      .then(setUser);
  }, [userId]);

  // Fetch user posts
  $effect(() => {
    if (!user) return;
    fetch(`/api/users/${userId}/posts`)
      .then((res) => res.json())
      .then(setPosts);
  }, [user]);

  // Setup notifications
  $effect(() => {
    if (!user) return;

    const subscription = subscribeToNotifications(userId, (newNotification) => {
      setNotifications((prev) => [...prev, newNotification]);
    });

    return () => subscription.unsubscribe();
  }, [user]);

  // ... render code
};

Conditional $effects ​

jsx
import { $effect, State } from "olova";

const ConditionalFetch = ({ shouldFetch, id }) => {
  const [data, setData] = State(null);

  $effect(() => {
    if (!shouldFetch) return;

    fetch(`/api/data/${id}`)
      .then((res) => res.json())
      .then(setData);
  }, [shouldFetch, id]);

  return shouldFetch ? <div>{data}</div> : null;
};

Best Practices ​

1. Keep $effects Pure ​

jsx
// ❌ Bad: Side $effect in condition
$effect(() => {
  if (data) {
    document.title = data.title;
  }
}, [data]);

// ✅ Good: Pure $effect with guard clause
$effect(() => {
  if (!data) return;
  document.title = data.title;
}, [data]);

2. Handle Race Conditions ​

jsx
import { $effect, State } from "olova";

const SearchResults = ({ query }) => {
  const [results, setResults] = State([]);

  $effect(() => {
    let isCurrent = true;

    const fetchResults = async () => {
      const response = await fetch(`/api/search?q=${query}`);
      const data = await response.json();

      if (isCurrent) {
        setResults(data);
      }
    };

    fetchResults();

    return () => {
      isCurrent = false;
    };
  }, [query]);

  // ... render code
};

3. Proper Dependency Management ​

jsx
// ❌ Bad: Missing dependency
const [count, setCount] = State(0);
$effect(() => {
  console.log(count);
}, []); // Missing count dependency

// ✅ Good: Including all dependencies
$effect(() => {
  console.log(count);
}, [count]);

4. $effect Organization ​

jsx
// ✅ Good: Split related $effects
$effect(() => {
  // Handle user authentication
}, [user]);

$effect(() => {
  // Handle user preferences
}, [preferences]);

// ❌ Bad: Mixing unrelated concerns
$effect(() => {
  // Handle both authentication and preferences
}, [user, preferences]);

Common Pitfalls ​

1. Infinite Loops ​

jsx
// ❌ Bad: Creates infinite loop
const [data, setData] = State([]);

$effect(() => {
  setData([...data, newItem]); // $effect updates state, causing re-render
}, [data]); // Which triggers $effect again

// ✅ Good: Use appropriate dependencies
$effect(() => {
  if (newItem) {
    setData((prev) => [...prev, newItem]);
  }
}, [newItem]);

2. Stale Closures ​

jsx
// ❌ Bad: Stale closure in interval
$effect(() => {
  const interval = setInterval(() => {
    console.log(count); // Captures initial count value
  }, 1000);

  return () => clearInterval(interval);
}, []); // Missing dependency

// ✅ Good: Include necessary dependency
$effect(() => {
  const interval = setInterval(() => {
    console.log(count);
  }, 1000);

  return () => clearInterval(interval);
}, [count]);

Performance Optimization ​

1. Debouncing $effects ​

jsx
import { $effect, State } from "olova";

const SearchInput = () => {
  const [query, setQuery] = State("");
  const [results, setResults] = State([]);

  $effect(() => {
    const timer = setTimeout(() => {
      if (query) {
        fetch(`/api/search?q=${query}`)
          .then((res) => res.json())
          .then(setResults);
      }
    }, 500);

    return () => clearTimeout(timer);
  }, [query]);

  return (
    <input
      value={query}
      onChange={(e) => setQuery(e.target.value)}
      placeholder="Search..."
    />
  );
};

Remember that $effects should be used for side $effects only. For computations that don't involve side $effects, consider using the Memo hook instead.
