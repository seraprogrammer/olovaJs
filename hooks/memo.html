$memo Hook Documentation ‚Äã

A comprehensive guide to using the $memo hook in Olova Framework for performance optimization

Introduction ‚Äã

The $memo hook is a powerful performance optimization tool that lets you $memoize expensive calculations to prevent unnecessary recomputations between renders. It returns a $memoized value that only changes when its dependencies change.

Basic Syntax ‚Äã

jsx
import { $memo } from "olova";

const $memoizedValue = $memo(
  () => computeExpensiveValue(a, b),
  [a, b] // dependency array
);

Core Concepts ‚Äã

Key Features ‚Äã

  • $memoizes return value of the calculation
  • Only recomputes when dependencies change
  • Helps prevent expensive calculations on every render
  • Returns same reference if dependencies haven't changed

Basic Examples ‚Äã

Simple Computation ‚Äã

jsx
import { $memo, $signal } from "olova";

const Calculator = () => {
  const [number, setNumber] = $signal(1);
  const [multiplier, setMultiplier] = $signal(2);

  const result = $memo(() => {
    // This calculation only runs when number or multiplier changes
    console.log("Calculating...");
    return number * multiplier;
  }, [number, multiplier]);

  return (
    <div>
      <input
        type="number"
        value={number}
        onChange={(e) => setNumber(Number(e.target.value))}
      />
      <input
        type="number"
        value={multiplier}
        onChange={(e) => setMultiplier(Number(e.target.value))}
      />
      <p>Result: {result}</p>
    </div>
  );
};

Filtered List ‚Äã

jsx
import { $memo, $signal } from "olova";

const FilteredList = ({ items }) => {
  const [filter, setFilter] = $signal("");

  const filteredItems = $memo(() => {
    console.log("Filtering items...");
    return items.filter((item) =>
      item.name.toLowerCase().includes(filter.toLowerCase())
    );
  }, [items, filter]);

  return (
    <div>
      <input
        value={filter}
        onChange={(e) => setFilter(e.target.value)}
        placeholder="Filter items..."
      />
      <ul>
        {filteredItems.map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
};

Advanced Usage ‚Äã

Complex Calculations ‚Äã

jsx
import { $memo, $signal } from "olova";

const DataAnalytics = ({ data }) => {
  const [metric, setMetric] = $signal("average");

  const statistics = $memo(() => {
    console.log("Computing statistics...");

    const values = data.map((item) => item.value);

    return {
      average: values.reduce((a, b) => a + b, 0) / values.length,
      max: Math.max(...values),
      min: Math.min(...values),
      sum: values.reduce((a, b) => a + b, 0),
      count: values.length,
    };
  }, [data]);

  const displayValue = $memo(() => {
    return statistics[metric].toFixed(2);
  }, [statistics, metric]);

  return (
    <div>
      <select value={metric} onChange={(e) => setMetric(e.target.value)}>
        <option value="average">Average</option>
        <option value="max">Maximum</option>
        <option value="min">Minimum</option>
        <option value="sum">Sum</option>
        <option value="count">Count</option>
      </select>
      <p>Result: {displayValue}</p>
    </div>
  );
};

Best Practices ‚Äã

1. Dependency Array Management ‚Äã

jsx
// ‚úÖ Good: All dependencies listed
const result = $memo(() => {
  return expensiveCalculation(prop1, prop2);
}, [prop1, prop2]);

// ‚ùå Bad: Missing dependencies
const result = $memo(() => {
  return expensiveCalculation(prop1, prop2);
}, [prop1]); // prop2 is missing

// ‚ùå Bad: Unnecessary dependencies
const result = $memo(() => {
  return expensiveCalculation(prop1);
}, [prop1, prop2]); // prop2 is not used

2. $memoization Granularity ‚Äã

jsx
// ‚úÖ Good: $memoize expensive calculations
const sortedFilteredItems = $memo(() => {
  return items
    .filter((item) => item.price > minPrice)
    .sort((a, b) => b.price - a.price);
}, [items, minPrice]);

// ‚ùå Bad: $memoizing simple calculations
const double = $memo(() => {
  return number * 2;
}, [number]); // Overkill for simple math

3. Nested $memos ‚Äã

jsx
const ComplexComponent = ({ data }) => {
  // First level of $memoization
  const processedData = $memo(() => {
    return heavyProcessing(data);
  }, [data]);

  // Second level using results from first
  const finalResult = $memo(() => {
    return furtherProcessing(processedData);
  }, [processedData]);

  return <div>{finalResult}</div>;
};

Common Pitfalls ‚Äã

1. Object and Array Dependencies ‚Äã

jsx
// ‚ùå Bad: New object reference on every render
const Component = () => {
  const options = { color: "red" };

  const result = $memo(() => {
    return expensiveCalculation(options);
  }, [options]); // Will rerun on every render

  return <div>{result}</div>;
};

// ‚úÖ Good: $memoize object or create outside component
const Component = () => {
  const options = $memo(
    () => ({
      color: "red",
    }),
    []
  ); // Empty deps if values don't change

  const result = $memo(() => {
    return expensiveCalculation(options);
  }, [options]);

  return <div>{result}</div>;
};

2. Function Dependencies ‚Äã

jsx
// ‚ùå Bad: Function recreation on every render
const Component = ({ data }) => {
  const processItem = (item) => {
    return item.value * 2;
  };

  const result = $memo(() => {
    return data.map(processItem);
  }, [data, processItem]); // processItem is recreated every render

  return <div>{result}</div>;
};

// ‚úÖ Good: Define stable function reference
const Component = ({ data }) => {
  const processItem = $memo(() => {
    return (item) => item.value * 2;
  }, []);

  const result = $memo(() => {
    return data.map(processItem);
  }, [data, processItem]);

  return <div>{result}</div>;
};

Performance Tips ‚Äã

1. When to Use $memo ‚Äã

jsx
// ‚úÖ Good use cases:
- Complex calculations
- Expensive filtering/sorting
- Data transformations
- Style object generation
- Complex string formatting

// ‚ùå Avoid using for:
- Simple calculations
- Direct prop passing
- Small arrays/objects
- Single value transformations

2. Combining with Other Hooks ‚Äã

jsx
const DataVisualizer = ({ data }) => {
  const [view, setView] = $signal("table");

  // Process data once
  const processedData = $memo(() => {
    return heavyDataProcessing(data);
  }, [data]);

  // Create view-specific data
  const viewData = $memo(() => {
    return formatDataForView(processedData, view);
  }, [processedData, view]);

  Effect(() => {
    // Use $memoized data in effects
    saveViewsignal(viewData);
  }, [viewData]);

  return <DataView data={viewData} />;
};

Remember:

  • Use $memo for expensive calculations only
  • Always specify the correct dependencies
  • Consider the cost of calculating vs. $memoizing
  • Combine with other hooks thoughtfully for optimal performance
