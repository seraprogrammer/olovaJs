$signal Hook Documentation ​

A comprehensive guide to using the $signal hook in Olova Framework

Introduction ​

The $signal hook is one of the fundamental hooks in Olova that allows you to add $signal management to your functional components. It provides a way to store and update component-specific data that can change over time.

Basic Syntax ​

jsx
import { $signal } from "olova";

// Basic usage
const [value, setValue] = $signal(initialValue);

Where:

  • initialValue: The initial $signal value
  • value: The current $signal value
  • setValue: A function to update the $signal value

Basic Examples ​

Counter Component ​

jsx
import { $signal } from "olova";

const Counter = () => {
  const [count, setCount] = $signal(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
    </div>
  );
};

Toggle Component ​

jsx
import { $signal } from "olova";

const Toggle = () => {
  const [isOn, setIsOn] = $signal(false);

  return (
    <div>
      <p>Status: {isOn ? "ON" : "OFF"}</p>
      <button onClick={() => setIsOn(!isOn)}>Toggle</button>
    </div>
  );
};

Working with Objects ​

Object $signal Management ​

jsx
import { $signal } from "olova";

const UserProfile = () => {
  const [user, setUser] = $signal({
    name: "",
    email: "",
    age: 0,
  });

  const updateField = (field, value) => {
    setUser({
      ...user,
      [field]: value,
    });
  };

  return (
    <div>
      <input
        value={user.name}
        onChange={(e) => updateField("name", e.target.value)}
        placeholder="Name"
      />
      <input
        value={user.email}
        onChange={(e) => updateField("email", e.target.value)}
        placeholder="Email"
      />
      <input
        type="number"
        value={user.age}
        onChange={(e) => updateField("age", parseInt(e.target.value))}
        placeholder="Age"
      />
    </div>
  );
};

Working with Arrays ​

Array $signal Management ​

jsx
import { $signal } from "olova";

const TodoList = () => {
  const [todos, setTodos] = $signal([]);
  const [input, setInput] = $signal("");

  const addTodo = () => {
    if (input.trim()) {
      setTodos([...todos, { id: Date.now(), text: input, completed: false }]);
      setInput("");
    }
  };

  const toggleTodo = (id) => {
    setTodos(
      todos.map((todo) =>
        todo.id === id ? { ...todo, completed: !todo.completed } : todo
      )
    );
  };

  const removeTodo = (id) => {
    setTodos(todos.filter((todo) => todo.id !== id));
  };

  return (
    <div>
      <input
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Add todo"
      />
      <button onClick={addTodo}>Add</button>

      <ul>
        {todos.map((todo) => (
          <li key={todo.id}>
            <input
              type="checkbox"
              checked={todo.completed}
              onChange={() => toggleTodo(todo.id)}
            />
            <span
              style={{
                textDecoration: todo.completed ? "line-through" : "none",
              }}
            >
              {todo.text}
            </span>
            <button onClick={() => removeTodo(todo.id)}>Delete</button>
          </li>
        ))}
      </ul>
    </div>
  );
};

Advanced Usage ​

Lazy Initial $signal ​

jsx
import { $signal } from "olova";

const ExpensiveInitialization = () => {
  // Use a function for expensive initial $signal calculations
  const [value, setValue] = $signal(() => {
    const expensiveComputation = () => {
      // Simulate expensive computation
      return Array(1000)
        .fill(0)
        .reduce((acc) => acc + 1, 0);
    };

    return expensiveComputation();
  });

  return <div>Computed Value: {value}</div>;
};

$signal with Previous Value ​

jsx
import { $signal } from "olova";

const Counter = () => {
  const [count, setCount] = $signal(0);

  const incrementByTwo = () => {
    // Using previous $signal value
    setCount((prevCount) => prevCount + 1);
    setCount((prevCount) => prevCount + 1);
  };

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={incrementByTwo}>Increment by 2</button>
    </div>
  );
};

Multiple $signals ​

jsx
import { $signal } from "olova";

const Form = () => {
  const [firstName, setFirstName] = $signal("");
  const [lastName, setLastName] = $signal("");
  const [email, setEmail] = $signal("");
  const [isValid, setIsValid] = $signal(false);

  const validateForm = () => {
    const valid = firstName.trim() && lastName.trim() && email.includes("@");
    setIsValid(valid);
  };

  return (
    <form>
      <input
        value={firstName}
        onChange={(e) => {
          setFirstName(e.target.value);
          validateForm();
        }}
        placeholder="First Name"
      />
      <input
        value={lastName}
        onChange={(e) => {
          setLastName(e.target.value);
          validateForm();
        }}
        placeholder="Last Name"
      />
      <input
        value={email}
        onChange={(e) => {
          setEmail(e.target.value);
          validateForm();
        }}
        placeholder="Email"
      />
      <button disabled={!isValid}>Submit</button>
    </form>
  );
};

Best Practices ​

  1. $signal Initialization

    • Always initialize $signal with a proper default value
    • Use lazy initialization for expensive computations
    • Keep $signal minimal and focused

  2. $signal Updates

    • Always treat $signal as immutable
    • Use the spread operator for object/array updates
    • Use functional updates when new $signal depends on previous $signal

  3. $signal Organization

    • Split complex $signal into multiple smaller $signals
    • Consider using objects for related $signal values
    • Keep $signal as close as possible to where it's used

  4. Common Pitfalls

    jsx
    // ❌ Wrong: Direct $signal mutation
const [user, setUser] = $signal({ name: "John" });
user.name = "Jane"; // Don't do this!

// ✅ Correct: Immutable update
setUser({ ...user, name: "Jane" });

// ❌ Wrong: Using $signal value right after setting it
const [count, setCount] = $signal(0);
setCount(count + 1);
console.log(count); // Will still show old value

// ✅ Correct: Use Effect or callback if you need the updated value
$effect(() => {
  console.log(count);
}, [count]);

Performance Considerations ​

  1. Avoid Unnecessary Updates

    jsx
    // Use callback function for expensive updates
const [data, setData] = $signal([]);

// Better performance for large arrays
const addItem = () => {
  setData((prevData) => [...prevData, newItem]);
};

  2. Batch Updates

    jsx
    const updateMultipleSignals = () => {
  setFirstName("John"); // These updates will be
  setLastName("Doe"); // batched together by
  setAge(25); // Olova automatically
};

Migration from Class Components ​

If you're migrating from class components, here's how $signal hook compares to traditional $signal:

jsx
// Class Component
class Counter extends Component {
  signal = { count: 0 };

  increment = () => {
    this.setsignal({ count: this.signal.count + 1 });
  };
}

// Functional Component with $signal Hook
const Counter = () => {
  const [count, setCount] = $signal(0);

  const increment = () => {
    setCount(count + 1);
  };
};

Remember that the $signal hook reinitializes the $signal on every render, but Olova ensures that only the setter function is updated while maintaining the $signal value between renders.
