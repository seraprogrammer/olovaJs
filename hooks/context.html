Context Hook Documentation ​

A comprehensive guide to using the Context hook in Olova Framework for $signal management and dependency injection

Introduction ​

The $context hook provides a way to pass data through the component tree without having to pass props manually at every level. It's useful for sharing data that can be considered "global" for a tree of components.

Basic Syntax ​

jsx
import { $context } from "olova";

// Create a $context
const Mycontext = $context.create(defaultValue);

// Context Provider
<MyContext.Provider value={value}>{children}</MyContext.Provider>;

// Use context in a component
const value = MyContext.use();

Core Concepts ​

Key Features ​

  • Share values across components
  • Avoid prop drilling
  • Update multiple components from a single source
  • Provide default values
  • Nested $context support

Basic Examples ​

Theme $context ​

jsx
import { $context, $signal } from "olova";

// Create theme $context
const ThemeContext = $context.create("light");

// Theme provider component
const ThemeProvider = ({ children }) => {
  const [theme, setTheme] = $signal("light");

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};

// Component using theme
const ThemedButton = () => {
  const { theme, setTheme } = ThemeContext.use();

  return (
    <button
      style={{
        background: theme === "light" ? "#fff" : "#333",
        color: theme === "light" ? "#333" : "#fff",
      }}
      onClick={() => setTheme(theme === "light" ? "dark" : "light")}
    >
      Toggle Theme
    </button>
  );
};

// App composition
const App = () => {
  return (
    <ThemeProvider>
      <ThemedButton />
    </ThemeProvider>
  );
};

User Authentication Context ​

jsx
import { $context, $signal, Effect } from "olova";

// Create auth context
const AuthContext = $context.create({
  user: null,
  login: () => {},
  logout: () => {},
});

// Auth provider component
const AuthProvider = ({ children }) => {
  const [user, setUser] = $signal(null);

  const login = async (credentials) => {
    try {
      const response = await fetch("/api/login", {
        method: "POST",
        body: JSON.stringify(credentials),
      });
      const userData = await response.json();
      setUser(userData);
    } catch (error) {
      console.error("Login failed:", error);
    }
  };

  const logout = () => {
    setUser(null);
  };

  return (
    <AuthContext.Provider value={{ user, login, logout }}>
      {children}
    </AuthContext.Provider>
  );
};

// Login component
const LoginForm = () => {
  const { login } = AuthContext.use();
  const [credentials, setCredentials] = $signal({
    username: "",
    password: "",
  });

  const handleSubmit = (e) => {
    e.preventDefault();
    login(credentials);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={credentials.username}
        onChange={(e) =>
          setCredentials({
            ...credentials,
            username: e.target.value,
          })
        }
      />
      <input
        type="password"
        value={credentials.password}
        onChange={(e) =>
          setCredentials({
            ...credentials,
            password: e.target.value,
          })
        }
      />
      <button type="submit">Login</button>
    </form>
  );
};

Advanced Usage ​

Multiple $contexts ​

jsx
import { $context, $signal } from "olova";

// Create multiple contexts
const LanguageContext = $context.create("en");
const ThemeContext = $context.create("light");
const UserContext = $context.create(null);

// App-level provider composition
const AppProvider = ({ children }) => {
  const [language, setLanguage] = $signal("en");
  const [theme, setTheme] = $signal("light");
  const [user, setUser] = $signal(null);

  return (
    <LanguageContext.Provider value={{ language, setLanguage }}>
      <ThemeContext.Provider value={{ theme, setTheme }}>
        <UserContext.Provider value={{ user, setUser }}>
          {children}
        </UserContext.Provider>
      </ThemeContext.Provider>
    </LanguageContext.Provider>
  );
};

// Component using multiple contexts
const UserProfile = () => {
  const { language } = LanguageContext.use();
  const { theme } = ThemeContext.use();
  const { user } = UserContext.use();

  return (
    <div style={{ background: theme === "light" ? "#fff" : "#333" }}>
      <h1>
        {language === "en" ? "Welcome" : "Bienvenido"}, {user?.name}
      </h1>
    </div>
  );
};

Context with Reducers ​

jsx
import { $context, $signal } from "olova";

// Create store context
const StoreContext = $context.create(null);

// Action types
const ACTIONS = {
  ADD_TODO: "ADD_TODO",
  TOGGLE_TODO: "TOGGLE_TODO",
  DELETE_TODO: "DELETE_TODO",
};

// Reducer function
const todoReducer = ($signal, action) => {
  switch (action.type) {
    case ACTIONS.ADD_TODO:
      return [
        ...$signal,
        { id: Date.now(), text: action.payload, completed: false },
      ];
    case ACTIONS.TOGGLE_TODO:
      return $signal.map((todo) =>
        todo.id === action.payload
          ? { ...todo, completed: !todo.completed }
          : todo
      );
    case ACTIONS.DELETE_TODO:
      return $signal.filter((todo) => todo.id !== action.payload);
    default:
      return $signal;
  }
};

// Store provider
const StoreProvider = ({ children }) => {
  const [todos, dispatch] = $signal([]);

  const value = {
    todos,
    addTodo: (text) => dispatch({ type: ACTIONS.ADD_TODO, payload: text }),
    toggleTodo: (id) => dispatch({ type: ACTIONS.TOGGLE_TODO, payload: id }),
    deleteTodo: (id) => dispatch({ type: ACTIONS.DELETE_TODO, payload: id }),
  };

  return (
    <StoreContext.Provider value={value}>{children}</StoreContext.Provider>
  );
};

Best Practices ​

1. Context Organization ​

jsx
// ✅ Good: Separate contexts by domain
const UserContext = $context.create(null);
const ThemeContext = $context.create("light");
const SettingsContext = $context.create({});

// ❌ Bad: Single context for everything
const AppContext = $context.create({
  user: null,
  theme: "light",
  settings: {},
  // ... many unrelated values
});

2. Default Values ​

jsx
// ✅ Good: Meaningful default values
const ThemeContext = $context.create({
  theme: "light",
  toggleTheme: () => {
    console.warn("ThemeContext not initialized");
  },
});

// ❌ Bad: No default value
const ThemeContext = $context.create();

3. Error Handling ​

jsx
const UserContext = $context.create(null);

const UserProfile = () => {
  const user = UserContext.use();

  if (!user) {
    return <div>Please log in to view profile</div>;
  }

  return <div>Welcome, {user.name}</div>;
};

Performance Optimization ​

1. $context Value $$memoization ​

jsx
import { $context, $signal, $memo } from "olova";

const SettingsProvider = ({ children }) => {
  const [theme, setTheme] = $signal("light");
  const [language, setLanguage] = $signal("en");

  // $memoize context value to prevent unnecessary re-renders
  const value = $memo(
    () => ({
      theme,
      language,
      setTheme,
      setLanguage,
    }),
    [theme, language]
  );

  return (
    <SettingsContext.Provider value={value}>
      {children}
    </SettingsContext.Provider>
  );
};

2. Context Splitting ​

jsx
// ✅ Good: Split frequently changing values
const ThemeContext = $context.create(null);
const UserContext = $context.create(null);

// Components only re-render when their specific context changes
const ThemeAwareComponent = () => {
  const theme = ThemeContext.use();
  return <div style={{ background: theme }}>...</div>;
};

const UserAwareComponent = () => {
  const user = UserContext.use();
  return <div>{user.name}</div>;
};

Common Pitfalls ​

1. Context Provider Missing ​

jsx
// ❌ Bad: Using context without provider
const Component = () => {
  const theme = ThemeContext.use(); // Will use default value
  return <div>{theme}</div>;
};

// ✅ Good: Ensure provider is present
const App = () => {
  return (
    <ThemeContext.Provider value="light">
      <Component />
    </ThemeContext.Provider>
  );
};

2. Unnecessary Context Usage ​

jsx
// ❌ Bad: Using context for props that could be passed directly
const Parent = () => {
  return <Child />;
};

const Child = () => {
  const theme = ThemeContext.use(); // Unnecessary if only Parent needs theme
  return <div>{theme}</div>;
};

// ✅ Good: Pass props directly for shallow component trees
const Parent = () => {
  const theme = ThemeContext.use();
  return <Child theme={theme} />;
};

const Child = ({ theme }) => {
  return <div>{theme}</div>;
};

Remember:

  • Use $context for truly global or widely shared $signal
  • Keep contexts focused and separated by domain
  • Provide meaningful default values
  • Consider performance implications of $context updates
  • Use Error Boundaries to handle $context-related errors
